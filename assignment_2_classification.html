import spacy
from tqdm import tqdm

# Load the spaCy model
nlp = spacy.load("en_core_web_sm")

# Provided clean_text function
def clean_text(corpus, spacy_pipeline, n_proc=2, batch_size=1000):
    """
    Cleans a sequence of text by applying some simple processing techniques.

    Args:
        corpus (Iterable): a sequence of text to be processed
        spacy_pipeline: the Spacy pipeline object for processing text
        n_proc (int): the number of processors to use for parallel processing
        batch_size (int): the number of texts to process in a single batch

    Returns:
        clean_sequence (list): a cleaned version of the original text
    """
    # container to store cleaned documents
    corpus_clean = []

    for doc in spacy_pipeline.pipe(
        corpus,
        disable=["ner"],
        n_process=n_proc,
        batch_size=batch_size
    ):

        # container for cleaned document tokens
        doc_tokens = [
            token.lemma_.lower()
            for token in doc
            if not token.is_stop and token.is_alpha
        ]

        # recombining tokens into a string
        clean_doc = " ".join(doc_tokens)
        
        corpus_clean.append(clean_doc)

    return corpus_clean

4 S0003682X1500242X Validation of the blocked force method for various boundary conditions for automotive source characterization Vibro-acoustic source characterization is an essential task in vehicle development to enable prediction of receiver response. For structure-borne noise, the interface forces in multiple degrees of freedom due to internal loads are often quantified for root cause analyses in a single system assembly, as in transfer path analysis (TPA). However, for a reliable prognosis of the acoustic performance of a known component such as a motor or pump, a receiver-independent source characterization is required, and the method of acquiring blocked forces from in-situ measurements has been shown to be a preferred technique for such purposes. The benefits of the method are the characterization of the intrinsic properties of the source and the possibilities of measuring the component attached to receivers with varying dynamic properties. There is to date a limited number of validation cases where blocked forces from in-situ measurements are acquired for automotive source-receiver assemblies. In this study the blocked forces of a vacuum pump in nine degrees of freedom were determined when connected to a bracket whose boundary conditions were modified in order to achieve four assemblies with different source/receiver dynamic properties. The results show that the blocked forces are transferable, i.e. the receiver response in one assembly was predicted in a wide frequency range by combining source-receiver transfer functions of that assembly with blocked forces estimated in another assembly. Furthermore, an in-situ blocked force TPA was applied to a double-isolated complete vehicle source-receiver case of an electric rear axle drive with interior compartment sound pressure as target. The reconstructed magnetic tonal harmonics agreed with the measured target response in the frequency range 50-500 Hz, which further motivates the use of the blocked force principles for TPA and source requirements specifications. phys_sci Validation of the blocked force method for various boundary conditions for automotive source characterization Vibro-acoustic source characterization is an essential task in vehicle development to enable prediction of receiver response. Evaluate and Compare Models

Based on "When searching, users have a preference for not being shown articles that are irrelevant to their search", precision is a crucial metric, then f1-score.

Multinomial Naive Bayes and Logistic Regression are generally fast for both training and inference.

SVM is slow to train, especially with large data, but moderately fast during inference.

Random Forest is slower to train but provides reasonable inference times.

Logistic Regression and Support Vector Machine (SVM) performs the best in terms of precision and f1-score.

Multinomial Naive Bayes (NB)

Strengths: Fast, handles high-dimensional text data well.

Shortcomings: Assumes feature independence, which leads to lower precision and recall in complex text patterns.

Logistic Regression

Strengths: Simple, interpretable, effective with high-dimensional, linearly separable data.

Shortcomings: Struggles with non-linear relationships and requires feature scaling.

Support Vector Machine

Strengths: Excellent for high-dimensional data, handles complex patterns well.

Shortcomings: Slow to train and requires careful hyperparameter tuning.

Random Forest

Strengths: Good for non-linear relationships, reduces overfitting.

Shortcomings: Less effective with high-dimensional sparse data like text, slower inference.

Logistic Regression and SVM performed best due to their ability to handle high-dimensional text data efficiently.
