def clean_text(corpus, spacy_pipeline, as_string=True, n_proc=2, batch_size=1000):
    """
    Cleans a sequence of text by applying some simple processing techniques.

    Args:
        corpus (Iterable): a sequence of text to be processed
        spacy_pipeline: the Spacy pipeline object for processing text
        n_proc (int): the number of processors to use for parallel processing
        batch_size (int): the number of texts to process in a single batch

    Returns:
        clean_sequence (list): a cleaned version of the original text
    """
    # container to store cleaned documents
    corpus_clean = []

    for doc in spacy_pipeline.pipe(
        corpus,
        disable=["ner"],
        n_process=n_proc,
        batch_size=batch_size
    ):

        # container for cleaned document tokens
        doc_tokens = [
            token.lemma_.lower()
            for token in doc
            if not token.is_stop and token.is_alpha and (len(token) > 1) and token.pos_ in ("NOUN", "VERB") # should consider if these are appropriate
        ]
        
        if as_string:
            corpus_clean.append(" ".join(doc_tokens))
        else:
            corpus_clean.append(doc_tokens)

    return corpus_clean

LDA tends to be more interpretable because it generates probabilistic topic distributions for both documents and words.

  1. Document-topic distribution: LDA assigns a probability distribution over topics for each document, showing how much each topic contributes to a documents content. This makes it easier to understand which topics are dominant in each document, especially when there are multiple topics.

  2. Topic-word distribution: LDA also provides a probability distribution of words for each topic, which helps identify the most relevant words that define each topic. The words with the highest probabilities make it clear what the topic is about.

  3. Coherence Score: LDA has built-in metrics like the coherence score to assess how meaningful and coherent the topics are. This helps in fine-tuning the model and improving interpretability.

In comparison, NMF only gives non-negative weights for topics and words, without the probabilistic interpretation, making it more difficult to clearly define the strength of topics in documents or assess topic coherence.

coherence_scores = []

for num_topics in topics_range:
    model = LdaMulticore(  # LDAMulitcore - can use multiple processes
        corpus=corpus,
        id2word=dictionary,
        chunksize=chunksize,
        alpha='symmetric', # idea alpha all three values are the same 
        eta='auto',
        iterations=iterations,
        num_topics=num_topics,
        passes=passes,
        eval_every=eval_every,
        workers=3,
        random_state=RANDOM_STATE
    )
    
    # Compute Coherence Score
    coherence_model = CoherenceModel(model=model, texts=normalized_corpus, dictionary=dictionary, coherence='c_v')
    coherence_score = coherence_model.get_coherence()
    coherence_scores.append(coherence_score)

NMF is a matrix factorization technique that approximates the original document-term matrix (like a TF-IDF matrix) into two non-negative matrices: one representing the topic weights for each document (W) and the other representing the word weights for each topic (H).

The values in W[doc_idx] represent the contribution of each topic to the document doc_idx, but they are not probabilities and are not required to sum to 1.

The values are just the weights indicating how much each topic contributes to that specific document, and they can take any non-negative values.

Model Results

My comparison between LDA and NMF highlights key differences in topic modeling approaches:

  • LDA (Latent Dirichlet Allocation): LDA provides a probabilistic approach, offering distinguishable topics with some overlap and leveraging tools like coherence score to determine the optimal number of topics. It tends to be slower but offers clearer metrics for topic quality, making it easier to tune the model. I achieved a higher coherence score with 8 topics and labeled them uniquely, which suggests LDA provided more meaningful and interpretable results for my dataset.

  • NMF (Non-Negative Matrix Factorization): NMF is a faster matrix factorization technique, allowing me to experiment with different numbers of topics quickly. However, as I noted, theres no direct metric like coherence score, making it harder to tune effectively. My experiment with 3, 8 and 12 topics showed that having too many/few topics can lead to "glued" tokens that are less comprehensible. NMF missed or misclassified some articles compared to LDA. So both LDA and NMF chose 8 topics.

Summary:

  • LDA was slower but provided more reliable topics with a higher coherence score.
  • NMF was faster and allowed more flexibility in experimenting with the number of topics, but it lacked the fine-tuned interpretability and comprehensiveness of LDA.

My analysis suggests that LDA might be more appropriate when topic coherence and detailed topic assignments are crucial, while NMF might be useful for quicker, large-scale experiments or when computational efficiency is important.

Challenges

When comparing LDA and NMF, both have some challenges:

  • LDA slower and more demanding on memory because it's a probabilistic model and requires multiple iterations (like Gibbs sampling) to estimate topic distributions. This makes it more computationally intensive, especially with larger datasets, which increases run time and memory usage.

  • NMF runs faster and is generally more memory-efficient since it uses matrix factorization instead of complex probabilistic calculations. Since theres no clear metric like coherence score, it requires more trial and error to find the right number of topics.

Overall, LDA takes more time and resources but often provides better interpretability, while NMF is quicker but needs more experimentation to fine-tune the topics.
