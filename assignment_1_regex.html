# Define a regex pattern to capture the regulation description and comments
pattern = r'\d+\.\s([A-Za-z\s,;&]+)\s-\sComments:\s(.+)'

[A-Za-z\s,;&]: This character class matches any uppercase or lowercase letter (A-Za-z), space (\s), comma (,), semicolon (;), or ampersand (&).

+: This means "one or more" of the above characters.

() (parentheses): These are used to create a capture group, which means the part matched by this regex will be stored for later use (for example, extract the regulation description using this group).

\s-\s it matches the - between the regulation description and the comments.

Comments:\s: This explicitly matches the "Comments:" followed by a space. By including this in the pattern but not in the capture group, it ensures that "Comments: " is recognized but not captured.

(.+): This remains the same to capture everything that comes after "Comments: " without including "Comments: " itself.

For the text:

  1. PHYSICAL FACILITIES INSTALLED, MAINTAINED & CLEAN - Comments: MUST REPAIR OR REPLACE STAINED CEILING TILES IN KIDS TOILET ROOM.

\d+ matches 55.

. matches the period after 55.

\s matches the space after the period.

([A-Za-z\s,;&]+) captures the regulation description: PHYSICAL FACILITIES INSTALLED, MAINTAINED & CLEAN.

\s-\s matches the - separating the description from the comments.

(.+) captures the comments: MUST REPAIR OR REPLACE STAINED CEILING TILES IN KIDS TOILET ROOM.

It Captures:

Group 1: The regulation description.

Group 2: The comments.

re.search() finds the first instance of a match anywhere in the string and returns a Match object. If no pattern is found, a None is returned.

# Extraction logic to handle both single and multiple violations
violations_data = []

for index, row in failed_inspections.iterrows():
    if pd.notna(row['Violations']):
        # If there's a '|' (multiple reasons), split and process each part
        if '|' in row['Violations']:
            violations = row['Violations'].split('|')
        else:
            # If no '|', treat it as a single violation
            violations = [row['Violations']]
        
        for violation in violations:
            match = re.search(pattern, violation.strip())  # Apply regex to each part # strip() method removes any leading, and trailing whitespaces.
            if match:
                regulation_description = match.group(1).strip()
                comments = match.group(2).strip()
                violations_data.append([regulation_description, comments])

# Function to preprocess comments
def preprocess_text(text):
    # Convert to lowercase
    text = text.lower()
    # Tokenize the text
    tokens = word_tokenize(text)
    # Remove punctuation and stopwords
    stop_words = set(stopwords.words('english'))
    tokens = [token for token in tokens if token not in string.punctuation and token not in stop_words]
    return tokens

# Helper function to convert Penn Treebank tags to WordNet POS tags
def get_wordnet_pos(treebank_tag, skip_value="n"):
    """Converts treebank tags to WordNet tags."""
    if treebank_tag.startswith('J'):
        return wordnet.ADJ
    elif treebank_tag.startswith('V'):
        return wordnet.VERB
    elif treebank_tag.startswith('N'):
        return wordnet.NOUN
    elif treebank_tag.startswith('R'):
        return wordnet.ADV
    else:
        return skip_value
    

# Function to lemmatize tokens with POS tagging
def lemmatize_with_pos(tokens):
    # Step 1: Get POS tags for each token
    pos_tags = nltk.pos_tag(tokens)
    # Step 2: Convert POS tags to WordNet POS format using the helper function
    pos_tags_wordnet = [(token, get_wordnet_pos(tag)) for token, tag in pos_tags]
    # Step 3: Lemmatize tokens with their corresponding POS tags
    lemmatized_tokens = [lemmatizer.lemmatize(token, pos=pos_tag) for token, pos_tag in pos_tags_wordnet]
    return lemmatized_tokens

5. Briefly describe the differences in various tokens from:

  1. Raw tokens
  2. Preprocessed tokens
  3. Preprocessed tokens + Porter stemmer
  4. Preprocessed tokens + Lancaster stemmer
  5. Preprocessed tokens + lemmatization

1. Raw Tokens

  • Description: These tokens are directly extracted from the text without any cleaning or preprocessing. They include punctuation, stopwords (common words like "and", "to", "the"), and case-sensitive variations.
  • Examples: Tokens like . (punctuation), "AND", "TO", and "THE" dominate the list.
  • Observation: Raw tokens include many non-informative words (stopwords) and punctuation, which dilute the meaningful content. This is why common words like "AND" and punctuation marks dominate the token counts.

2. Preprocessed Tokens

  • Description: Preprocessed tokens have undergone cleaning, which includes converting text to lowercase, removing punctuation, and eliminating stopwords. This step reduces noise and keeps only meaningful words.
  • Examples: Tokens such as "observed", "instructed", "food", and "must" now appear at the top.
  • Observation: After preprocessing, the focus is shifted to more informative content, and tokens represent meaningful words that contribute to the understanding of the violation context.

3. Preprocessed Tokens + Porter Stemmer

  • Description: Porter stemming reduces words to their root form by removing common suffixes. Porter is relatively conservative and tries to balance reducing the word form while maintaining readability.
  • Examples: Words like "observed" become "observ", "instructed" becomes "instruct", and "violation" becomes "violat".
  • Observation: Stemming produces shorter, truncated versions of words. The tokens are often not real words but close to the root, which can improve some tasks but may lose clarity. For example, "priority" becomes "prioriti", and "violation" becomes "violat".

4. Preprocessed Tokens + Lancaster Stemmer

  • Description: Lancaster stemming is a more aggressive stemming method compared to Porter, often producing shorter stems.
  • Examples: "observed" becomes "observ", "instructed" becomes "instruct", and "priority" becomes "pri".
  • Observation: Lancaster stemmer produces even more truncated tokens than Porter. This aggressive reduction may make tokens less recognizable or readable but creates a consistent root for different word forms. For example, "priority" is reduced to "pri", which might be too aggressive in certain contexts.

5. Preprocessed Tokens + Lemmatization

  • Description: Lemmatization reduces words to their base (dictionary) form by considering the word's part of speech (POS). Unlike stemming, lemmatization retains meaningful word forms.
  • Examples: Words like "observed" become "observe", "instructed" becomes "instruct", and "violation" stays as "violation".
  • Observation: Lemmatization is more accurate and human-readable than stemming, as it converts words to their base form without excessive truncation. It provides a balance between reducing variations of a word and maintaining readability. For instance, "violation" remains "violation" rather than being reduced to a stem like "violat".

Key Differences:

  • Raw Tokens: Contain stopwords and punctuation, with minimal value for analysis.
  • Preprocessed Tokens: Cleaned and more focused on meaningful content, but words remain unaltered.
  • Porter Stemmer: Produces root forms of words, truncating words moderately.
  • Lancaster Stemmer: More aggressive in truncating words, which may sometimes result in overly shortened tokens.
  • Lemmatization: Produces base forms of words based on their POS, maintaining more readable and accurate tokens compared to stemming.

Overall, I would choose Preprocessed Tokens + Lemmatization
